# UI Content Rules

작성일: 2026-03-10

## 1. 목적

이 문서는 화면별 콘텐츠 표기 규칙을 고정하기 위한 초안이다.

## 2. 날짜 / 시간 포맷

## 2.1 공지 리스트 날짜

- 포맷: `YYYY-MM-DD`
- 예: `2024-03-15`

## 2.2 커뮤니티 게시글/채팅 시간

- 포맷: `3월 20일 오후 12:30`
- 채팅방 리스트처럼 짧게 쓸 때는 `오후 12:45` 형태 사용 가능

## 2.3 택시 출발 시간

- 포맷: `3월 20일 오전 09:00`

## 2.4 일정 기간 표기

- 포맷: `2024-05-10 ~ 2024-05-11`

## 2.5 시간표 날짜 헤더

- 포맷: `3월 10일 화요일 2주차`

## 3. 숫자 포맷

## 3.1 금액

- 천 단위 콤마 사용
- suffix: `원`
- 예: `3,500원`

## 3.2 인원

- 택시: `2/4명`
- 채팅방 인원: `1,247명`

## 3.3 배지 숫자

- 숫자만 표시
- 현재 상위 Figma에는 최대치 규칙이 보이지 않는다
- `99+` 같은 상한 표기는 별도 확정 필요

## 4. 라벨 규칙

## 4.1 카테고리 라벨

현재 상위 Figma에서 확인된 라벨은 아래와 같다.

- 학사
- 장학
- 취업
- 행사
- 시설
- 자유게시판
- 정보게시판
- 질문게시판

## 4.2 상태 라벨

- 모집중
- 마감
- 읽지 않은 공지
- 전체보기

## 5. 말줄임 규칙

## 5.1 공지

- Campus 공지 preview 제목: 1줄
- Notice 메인 리스트 제목: 1줄 우선

## 5.2 게시글

- 제목: 1줄
- 본문 미리보기: 1~2줄

## 5.3 채팅방

- 방 이름: 1줄
- 마지막 메시지: 1줄

## 5.4 택시

- 출발지/목적지: 1줄
- 요금: 줄바꿈 없이 1줄

## 6. 문구 톤

- 짧고 정보 중심적일 것
- 불필요한 감탄형 문구를 줄일 것
- 액션 라벨은 명사보다 동작 중심으로 작성
- 예: `보기`, `전체보기`, `프로필 수정`, `새 파티 만들기`

## 7. 빈 상태 / 에러 상태 문구

상위 Figma에는 명시된 예시가 부족하므로 다음 원칙만 우선 고정한다.

- 한 문장으로 현재 상태를 설명
- 바로 다음 행동을 함께 제시
- 비난형 문구 금지

세부 카피는 구현 직전 별도 확정이 필요하다.

## 8. 아이콘 의미 규칙

- 아이콘은 텍스트를 대체하지 않는다
- 카테고리 의미 전달보다 구조 보조 역할을 우선한다
- color만으로 상태를 구분하지 말고 라벨/텍스트를 함께 둔다

## 9. 접근성 관련 콘텐츠 규칙

- badge 숫자는 접근성 라벨에서 의미를 풀어 읽어야 한다
- 날짜/시간은 음성 읽기 시 혼동되지 않도록 포맷 일관 유지
- 축약어보다 완전한 한국어 라벨을 우선 사용한다
