# SKURI UI 리디자인 테마 분석 및 수정 계획

작성일: 2026-03-10  
작성 기준: 제공된 Figma 상위 프레임 7종(`SKURI-v2.0`) + 현재 `main` 코드베이스

## 1. 문서 목적

이 문서는 Figma MCP로 확인한 상위 화면 디자인을 기준으로, 현재 앱 UI를 어떤 방향으로 리팩터링해야 하는지 정리한 설계 문서다.

이번 문서는 다음 두 가지를 목표로 한다.

1. 제공된 Figma 화면에서 공통 UI 테마와 규칙을 추출한다.
2. 현재 코드베이스와의 차이를 기준으로 화면 개편 작업 계획을 정의한다.

## 3. Figma에서 확인한 전체 UI 테마

## 3.1 시각 방향

현재 코드베이스는 다크 테마 중심이지만, Figma는 전면적인 라이트 테마를 사용한다.

- 기본 배경: 매우 연한 그레이 `#F9FAFB`
- 카드/패널: 화이트 `#FFFFFF`
- 기본 텍스트: 짙은 슬레이트 `#111827`
- 보조 텍스트: `#4B5563`, `#6B7280`, `#9CA3AF`
- 핵심 포인트 컬러: 그린 `#22C55E`
- Campus 워드마크 컬러: 에메랄드 `#10B981`
- 보조 포인트 컬러:
  - 블루 `#2563EB`
  - 오렌지 `#FB923C` / `#EA580C`
  - 퍼플 `#9333EA`
  - 레드 `#EF4444`

핵심 인상은 다음과 같다.

- 검은 배경 기반의 강한 대비가 아니라, 밝고 가벼운 캠퍼스형 UI
- 과한 장식보다 카드와 칩, 라벨 컬러로 정보 성격을 구분
- 상단 헤더와 하단 탭바는 화이트 고정, 본문만 소프트 그레이 배경
- Campus는 일반 텍스트 타이틀 대신 그린 워드마크 `SKURI`를 사용한다.

## 3.2 타이포그래피 규칙

Figma 전반에서 보이는 기준은 아래와 같다.

- Campus 브랜드 워드마크: `24 / 32`, script display
- 화면 헤더 타이틀: `18 / 28`, bold
- My 프로필 이름: `20 / 28`, bold
- 섹션 타이틀: `16 / 24`, bold
- 본문/버튼 기본: `14 / 20`, regular 또는 medium
- 메타 정보/칩: `12 / 16`
- 탭 라벨/하단 탭 라벨: `10 / 15`

현재 `src/constants/typhograpy.ts`는 다크 테마용 스케일과 line-height가 섞여 있고, Figma 기준과 완전히 일치하지 않는다. 이번 개편에서는 라이트 테마와 함께 타이포 토큰도 재정의가 필요하다.

## 3.3 공통 컴포넌트 규칙

### 카드

- 주 카드 반경: `16`
- 소형 통계 카드/액션 패널 반경: `12`
- 배경은 대부분 화이트
- 그림자는 매우 약한 단일 그림자
  - 예: `0 1 2 rgba(0,0,0,0.05)`
- 경계선은 강하게 쓰지 않고, 필요한 곳만 `#E5E7EB` 또는 `#F3F4F6`

### 칩/필터

- 선택 상태: 그린 채움 + 화이트 텍스트
- 비선택 상태: 연한 회색 배경 `#F3F4F6`
- 반경은 풀 라운드
- 높이는 대체로 `36`

### 배지/카테고리 라벨

- 4px 반경의 작은 라벨
- 카테고리별 색 분리
  - 학사: 블루 계열
  - 장학: 퍼플 계열
  - 취업: 오렌지 계열
  - 행사: 핑크/레드 계열
  - 시설: 뉴트럴 그레이 계열
- 단, Campus의 읽지 않은 공지 preview는 카테고리별 색보다 `읽지 않음` 상태를 우선해 그린 계열로 단순화된다.

### 배너

- Notice 상단에는 읽지 않은 공지 요약 배너가 별도 카드로 배치된다.
- 배경은 soft green, 내부 CTA는 white pill button이다.
- 이 배너는 일반 리스트 카드와 다른 상태성 컴포넌트로 분리하는 것이 맞다.

### 헤더

- 높이: 실질 영역 `56`
- 배경: 화이트
- 하단 보더: `#E5E7EB`
- Campus: 브랜드 워드마크 + 프로필 단일 액션
- Taxi: 타이틀 + 프로필 단일 액션
- Notice / Community: 타이틀 + 검색 + 프로필
- My: 타이틀 only

### 세그먼트

- Community는 header 아래 고정 높이 세그먼트 탭을 둔다.
- 활성 상태는 green text + `2px` 하단 보더
- 비활성 상태는 tertiary text

### 하단 탭바

- 화이트 배경 + 상단 보더
- 활성 아이콘/라벨만 그린
- 비활성은 회색
- 현재 5탭 구조가 아니라 4탭 구조

## 3.4 화면별 성격

### Campus

- 홈이 아니라 `오늘 필요한 정보 허브`
- 인사형 홈이 아니라 브랜드 워드마크 기반 대시보드
- 긴 기능 나열보다 요약 카드 묶음 구조
- 섹션 순서가 명확하다:
  - 읽지 않은 공지
  - 오늘 시간표
  - 모집 중인 택시
  - 오늘의 학식
  - 다가오는 일정
  - 빠른 메뉴
- 시간표는 기본 접힘 상태와 야간수업 확장 상태를 모두 고려해야 한다.
- 택시는 전체 카드가 아니라 1줄 preview item 반복으로 축약된다.

### Taxi

- 가장 행동 중심적인 화면
- 상단 지도/검색 히어로 영역 + 필터칩 + 주요 CTA + 파티 카드 목록
- hero는 실제 지도 컨트롤보다 `soft green background + search field` 인지가 우선이다.
- 카드 안에서 상태 배지, 출발 시간, 경로, 인원, 요금이 한눈에 보여야 함

### Notice

- 정보 소비 효율이 우선
- 상단 배너와 필터칩 이후, 고밀도 리스트형 공지 카드
- 큰 카드형 피드보다 스캔 가능한 리스트 구조에 가까움
- Notice 메인 리스트는 여러 장의 카드가 아니라 `grouped list 1장 + row 반복`에 가깝다.

### Community

- 게시글 피드 기반
- 게시판/채팅 세그먼트 구조
- 인기 게시글은 별도 강조 카드
- 일반 게시글은 메타 정보가 압축된 피드 카드
- 글쓰기 FAB가 명확하게 분리됨
- 채팅 세그먼트는 피드형이 아니라 압축된 대화방 리스트 구조
- 채팅방 카드는 `아이콘 원형 아바타 + 방 이름 + 마지막 메시지 + 인원 수 + 시간 + 안읽음 배지` 조합
- 채팅 카드는 게시글보다 더 낮은 정보 밀도와 더 빠른 스캔 속도를 우선

### My

- 글로벌 보조 화면
- 프로필 요약 > 활동 통계 > 활동 메뉴 > 설정 메뉴 > 로그아웃
- 설정/활동은 동일한 흰색 그룹 리스트 패턴을 재사용
- 80px gradient avatar, 20px 이름, small green edit button이 상단 identity block을 이룬다.
- 실제 앱 shell에서는 bottom tab를 노출하지 않고, 좌측 상단 뒤로가기 버튼을 둔다.

### Minecraft 관련 정책

- 현재 코드에 존재하더라도 새 IA에 즉시 편입하지 않는다.
- 이번 리팩터링에서는 상위 화면에 노출하지 않는다.
- 정책 확정 전까지는 기능을 유지하되, 메인 허브와 내비게이션 구조 설계 기준에서는 제외한다.

### Campus 브랜드 자산 정책

- Campus 상단 `SKURI`는 텍스트 스타일링이 아니라 SVG/이미지 자산으로 구현한다.
- 좌측 브랜드 영역은 추후 로고/앱 아이콘 결합을 수용할 수 있게 설계한다.
- 단, 로고 조합 자체는 아직 확정하지 않는다.

## 4. 현재 코드베이스와의 핵심 차이

## 4.1 테마 방향이 반대다

현재 토큰은 다크 테마 기준이다.

- `COLORS.background.primary = #18181B`
- `COLORS.background.card = #232326`
- `COLORS.text.primary = #F3F4F6`

Figma는 라이트 테마 기준이므로, 부분 수정이 아니라 토큰 체계 자체를 뒤집어야 한다.

## 4.2 탭 구조가 다르다

현재 `src/navigations/types.ts` 기준 탭 구조는 다음과 같다.

- `홈`
- `택시`
- `공지`
- `게시판`
- `채팅`

Figma 기준 구조는 다음과 같다.

- `캠퍼스`
- `택시`
- `공지`
- `커뮤니티`
- `MY`는 탭이 아니라 글로벌 진입

즉, 네비게이션 레벨에서 다음 작업이 필요하다.

- `홈` -> `캠퍼스` 리브랜딩
- `게시판` + `채팅` -> `커뮤니티` 통합
- `Profile/Setting` 계열을 글로벌 MY 진입과 정렬

## 4.3 HomeScreen의 정보 우선순위가 Figma와 다르다

현재 `src/screens/HomeScreen.tsx`는 다음 순서다.

- 택시
- 공지
- 시간표
- 학사일정
- Minecraft
- 학식

Figma Campus는 택시 우선 허브가 아니라 `공지/시간표/오늘성 정보 중심 허브`다. 특히 Minecraft는 상위 허브의 핵심 모듈로 보이지 않는다.

## 4.4 Community가 분리 구현돼 있다

현재는 `BoardScreen.tsx`와 `ChatListScreen.tsx`가 완전히 분리돼 있다.

- Board는 별도 탭
- Chat은 별도 탭
- 상단 구조와 카드 패턴도 서로 다름

Figma는 하나의 `커뮤니티` 탭 안에 `게시판 / 채팅` 세그먼트를 두는 방향이다.

## 4.5 ProfileScreen 구조가 Figma My와 다르다

현재 `ProfileScreen.tsx`는 다크 테마 기반의 상세 프로필 화면 성격이 강하다.

- 뒤로가기 포함 전용 페이지
- 계좌/설정만 우선 노출
- 활동 통계 카드, grouped list, 로그아웃 CTA 구조가 Figma와 다름

Figma My는 글로벌 목적지형 정보 구조다.

## 5. 리팩터링 설계 원칙

## 5.1 우선순위

이번 UI 리팩터링은 단순 스타일 변경이 아니라 아래 순서로 접근해야 한다.

1. 토큰과 공통 컴포넌트 재정의
2. 하단 탭/헤더 구조 개편
3. 상위 5개 화면 리팩터링
4. 상세 화면 전파

토큰 정리 없이 화면부터 바꾸면 중복 스타일이 폭증할 가능성이 높다.

## 5.2 토큰 재정의 원칙

새 토큰 레이어는 최소 아래 단위로 재구성한다.

- `background`
  - `app`
  - `surface`
  - `elevated`
  - `subtle`
- `text`
  - `primary`
  - `secondary`
  - `tertiary`
  - `inverse`
- `border`
  - `default`
  - `subtle`
  - `strong`
- `accent`
  - `primaryGreen`
  - `blue`
  - `orange`
  - `purple`
  - `red`
- `status/category`
  - notice category colors
  - taxi state colors

타이포그래피도 `screenTitle / sectionTitle / body / meta / tabLabel` 수준으로 재정의하는 편이 구현과 유지보수 모두 유리하다.

## 5.3 공통 프리미티브 우선 제작

반복 사용될 가능성이 높은 공통 컴포넌트를 먼저 정리한다.

- AppHeader
- BottomTabItem style wrapper
- SectionHeader
- FilterChip
- StatusBadge
- CategoryTag
- ElevatedCard
- GroupedList / GroupedListItem
- FloatingActionButton

## 6. 화면별 수정 계획

## 6.1 Campus

### 목표

- 홈을 허브형 정보 화면으로 전환
- 기존 기능 나열형 스크롤을 오늘성 중심 구조로 재배치

### 작업 항목

- 상단 헤더를 인사형이 아니라 `브랜드 워드마크 + 프로필` 구조로 전환
- 읽지 않은 공지 요약 카드를 최상단 배치
- 시간표 카드를 핵심 카드로 승격
- 택시/학식/학사일정은 미리보기 모듈로 축소
- 시간표는 기본 접힘 상태와 야간수업 확장 상태를 모두 반영
- 빠른 메뉴 4개를 독립 아이콘 카드로 구성

### 코드 영향 예상

- `src/screens/HomeScreen.tsx`
- `src/components/home/*`
- `src/components/timetable/*`
- `src/components/cafeteria/*`
- `src/components/academic/*`

## 6.2 Taxi

### 목표

- 지도 영역, 필터칩, CTA, 파티 카드의 정보 위계 명확화

### 작업 항목

- 상단 지도/검색 영역을 `soft green hero + search field` 레이아웃으로 정렬
- 필터칩 스타일과 파티 생성 CTA 재구성
- 파티 카드를 공통 카드 시스템 위로 재작성
- 상태 배지, 경로 라인, 인원/요금 메타 영역 구조화

### 코드 영향 예상

- `src/screens/TaxiScreen.tsx`
- `src/screens/TaxiTab/components/*`
- `src/screens/TaxiTab/hooks/useTaxiScreenPresenter.ts`

## 6.3 Notice

## 8. 구현 전 추가 문서화 권장 항목

사용자가 이미 정리하려는 항목은 다음과 같다.

- 디자인 시스템
- 공통 컴포넌트
- 컬러 팔레트
- 전체 UI 톤 앤 무드

여기에 더해 아래 항목까지 문서화해 두는 편이 실제 구현 품질에 직접 도움이 된다.

1. 타이포그래피 스케일
   - 화면 헤더, 섹션 헤더, 카드 타이틀, 본문, 메타, 탭 라벨까지 어떤 스타일을 쓰는지

2. Spacing 시스템
   - 4, 8, 12, 16, 24 같은 기본 간격 단위와 섹션 간 간격 규칙

3. Radius / Border / Shadow 규칙
   - 카드, 칩, FAB, 모달, grouped list가 각각 어떤 반경과 그림자를 쓰는지

4. 상태 표현 규칙
   - 선택, 비선택, 비활성, 로딩, 에러, 빈 상태, 안읽음, 마감, 모집중 같은 상태를 어떻게 표현하는지

5. 카테고리/도메인 색상 매핑
   - 학사, 장학, 취업, 행사, 시설, 택시 상태, 커뮤니티 타입 등의 컬러 매핑

6. 네비게이션 패턴
   - 상단 헤더 액션, 글로벌 MY 진입, 탭 전환, 세그먼트 전환, FAB 사용 규칙

7. 리스트/피드 템플릿
   - 공지 리스트, 게시글 피드, 채팅방 리스트, grouped list처럼 반복될 레이아웃 템플릿

8. 아이콘 사용 규칙
   - 아이콘 세트, 크기 단계, 배경 원형 아바타/아이콘 배경색 규칙

9. 모션 규칙
   - 탭 전환, 세그먼트 전환, 화면 진입, FAB, 배지 업데이트에 애니메이션을 어디까지 허용할지

10. 콘텐츠 규칙
   - 한 줄/두 줄 말줄임, 날짜 포맷, 숫자 포맷, 라벨 길이 제한

11. 접근성 기준
   - 최소 터치 영역, 대비, Dynamic Type 대응 범위, 스크린리더 라벨링 원칙

12. Safe Area / 디바이스 대응 규칙
   - 상단 헤더, 하단 탭바, 스크롤 컨테이너, FAB 위치의 기준값

위 항목까지 먼저 정리해 두면, 구현 단계에서 "컴포넌트는 만들었지만 화면마다 조금씩 다르게 생기는 문제"를 크게 줄일 수 있다.
## 9. 확인 필요 사항

아래 항목은 현재 Figma 링크만으로는 확정할 수 없다. 구현 전에 사용자 확인이 필요하다.

1. 상세 화면 리디자인 범위
   - 이번 작업을 상위 화면 중심으로 한정할지
   - 아니면 공지 상세, 게시글 상세/작성, 택시 채팅/정산, 시간표 상세까지 한 번에 확장할지 확인 필요

2. 글로벌 MY 진입 정책
   - 모든 상위 화면 우상단 프로필 아이콘이 동일하게 MY로 연결되는지
   - 일부 화면은 검색/알림/프로필 조합을 유지할지 확인 필요

## 10. 현재 결론

제공된 Figma 기준으로 보면, 이번 리팩터링의 본질은 아래 세 가지다.

- 다크 테마 기반 5탭 앱을 라이트 테마 기반 4탭 + 글로벌 MY 구조로 재설계하는 것
- 홈을 기능 나열형 화면에서 캠퍼스 허브로 전환하는 것
- 게시판/채팅을 커뮤니티 체계로 재구성하는 것

위 방향 자체는 충분히 명확하다. 현재 기준으로는 `상위 화면 우선 구현`, `커뮤니티 채팅 포함`, `Minecraft는 메인 IA에서 보류`까지는 확정 가능하다. 남은 쟁점은 상세 화면 확장 시점과 글로벌 MY 진입 방식의 일관성이다.
