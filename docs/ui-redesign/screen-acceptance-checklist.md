# Screen Acceptance Checklist

작성일: 2026-03-10

## 1. 목적

상위 화면 구현이 어느 수준이면 완료로 볼 수 있는지 공통 기준을 정의한다.

## 2. 사용 방식

각 화면은 아래 순서로 상태를 본다.

1. 내부 구현 완료
2. Figma 대조 완료
3. 사용자 리뷰 대기
4. 사용자 승인 또는 추후 phase 보류

즉, 내부 기준을 모두 충족해도 사용자가 아직 종료하지 않았으면 최종 완료로 보지 않는다.

## 3. 공통 완료 기준

아래 항목은 `Campus / Taxi / Notice / Community / My` 모두에 공통으로 적용한다.

### 3.1 구조

- 상위 shell 구조가 해당 Figma 프레임과 일치한다.
- 헤더, 본문, 그리고 해당 화면에 존재하는 bottom tab / segment / FAB 위치가 문서 기준과 일치한다.
- 범위 밖 depth 2+ 화면 구조를 임의로 끼워 넣지 않는다.

### 3.2 시각 언어

- v2 token만 사용한다.
- v2 primitive를 우선 사용한다.
- 컬러 위계, 타이포 위계, 간격, radius, shadow가 문서 기준과 일치한다.

### 3.3 정보 위계

- 사용자가 첫 스캔에서 핵심 정보가 무엇인지 바로 알 수 있다.
- 강조 정보와 보조 정보가 뒤바뀌지 않는다.
- 텍스트 truncation, 날짜/시간/금액 포맷이 문서 기준과 일치한다.

### 3.4 상태 / 인터랙션

- selected, unread, active, disabled 등 주요 상태가 문서 기준과 일치한다.
- 세그먼트, 필터칩, FAB, 뒤로가기, 주요 CTA 동작이 맞다.
- 아직 정책이 없는 상태를 임의로 확정하지 않는다.

### 3.5 범위 통제

- 화면 구현 때문에 레거시 전체 구조를 함께 재작성하지 않는다.
- 해당 phase 범위를 벗어난 상세 화면까지 확장하지 않는다.
- Minecraft 등 보류 기능을 임의로 편입하지 않는다.

### 3.6 품질

- lint 기준을 통과한다.
- 관련 문서와 코드가 충돌하지 않는다.

## 4. 화면별 완료 기준

## 4.1 Campus

- 상단 브랜드 헤더가 `SKURI` 자산 + 프로필 진입 구조로 구현된다.
- 읽지 않은 공지, 시간표, 모집 중인 택시, 오늘의 학식, 다가오는 일정, 빠른 메뉴 6개 모듈이 모두 존재한다.
- 시간표는 기본 상태와 확장 상태를 모두 지원한다.
- 택시 모듈은 메인 카드가 아니라 compact preview로 구현된다.

## 4.2 Taxi

- hero, 필터칩, 메인 CTA, 파티 리스트 구조가 맞다.
- 카드에서 `시간 / 상태 / 경로 / 인원 / 요금` 위계가 즉시 읽힌다.
- 메인 CTA가 화면 상단부에서 명확히 인지된다.

## 4.3 Notice

- 상단 unread banner가 구현된다.
- 카테고리 필터칩 row가 구현된다.
- 리스트는 grouped list형 row 반복 구조를 따른다.
- category, date, unread, title 위계가 바로 구분된다.

## 4.4 Community

- `게시판 / 채팅` top segment가 구현된다.
- 게시판은 `인기 게시글 + 일반 피드` 구조를 가진다.
- 채팅은 room list 구조를 가진다.
- FAB는 게시판에서만 노출된다.

## 4.5 My

- bottom tab를 노출하지 않는다.
- 좌측 상단 뒤로가기 버튼이 있다.
- 프로필 identity block, 통계 카드, `내 활동`, `설정`, 로그아웃 구조가 구현된다.
- grouped list 패턴이 일관되게 재사용된다.

## 5. 사용자 승인 게이트

상위 화면은 아래 둘 중 하나가 있어야 최종 종료로 본다.

- 사용자가 명시적으로 `이 화면은 됐다`, `여기서 종료`, `이 버전으로 간다`라고 승인
- 사용자가 `이 화면은 다음 phase에서 다시 보자`, `일단 보류`라고 명시

즉, UI 화면은 내부 완료 기준만으로 자동 종료하지 않는다.

## 6. 상태 라벨 권장

문서나 작업 로그에서는 아래 상태를 권장한다.

- `internal-complete`
  - 내부 기준과 lint, Figma 대조까지 끝남
- `awaiting-review`
  - 사용자 확인 대기
- `approved`
  - 사용자 승인 완료
- `deferred`
  - 추후 phase로 보류
