# UI Navigation And IA

작성일: 2026-03-10

## 1. 최종 상위 구조

상위 IA는 아래 구조를 기준으로 한다.

```text
캠퍼스
택시
공지
커뮤니티
+ 글로벌 MY
```

## 2. 현재 구조와 목표 구조

| 현재 | 목표 |
| --- | --- |
| 홈 | 캠퍼스 |
| 택시 | 택시 |
| 공지 | 공지 |
| 게시판 | 커뮤니티 |
| 채팅 | 커뮤니티 내부 세그먼트 |
| 프로필/설정 | 글로벌 MY |

## 3. 탭별 책임

## 3.1 캠퍼스

- 오늘 필요한 정보 요약 허브
- 공지, 시간표, 택시, 학식, 일정, 빠른 메뉴 진입점
- 상세 소비는 각 전용 영역으로 위임

## 3.2 택시

- 파티 탐색
- 파티 생성
- 파티 참여/상태 확인

## 3.3 공지

- 공지 목록 소비
- 읽지 않은 공지 확인
- 카테고리 기반 탐색

## 3.4 커뮤니티

- 게시판
- 채팅
- 커뮤니티 탐색 허브

## 3.5 글로벌 MY

- 프로필
- 활동 요약
- 설정
- 계좌 관리
- 로그아웃
- 상단 좌측 뒤로가기

## 4. Community 구조

커뮤니티 탭은 내부 세그먼트로 나뉜다.

```text
커뮤니티
├─ 게시판
└─ 채팅
```

### 게시판

- 인기 게시글 강조
- 일반 게시글 피드
- FAB 기반 글쓰기

### 채팅

- 공개 채팅방 리스트
- 마지막 메시지와 안읽음 상태 중심

## 5. 글로벌 MY 정책

다음 두 가지는 확정됐다.

- MY는 하단 탭에 두지 않는다.
- 별도 목적지 화면으로 운영한다.
- MY 화면에서는 bottom tab shell을 노출하지 않는다.
- MY 화면 좌측 상단에는 뒤로가기 버튼을 둔다.

다음은 아직 보류다.

- 모든 상위 화면 우상단에서 동일하게 MY로 진입할지
- 검색/알림/프로필 조합을 화면별로 다르게 둘지

즉, `MY는 별도 화면`까지는 확정이고, `정확한 진입 버튼 표준`은 미정이다.
또한 최신 Figma My 프레임은 bottom tab를 함께 보여주지만, 실제 앱 shell 정책은 `bottom tab 미노출 + 좌측 상단 뒤로가기`를 기준으로 구현한다.

## 6. 상위 화면 우선 구현 정책

이번 구현은 아래 순서를 따른다.

1. 디자인 시스템
2. 공통 컴포넌트
3. 상위 화면
4. 상세 화면

즉, 상세 화면 리팩터링은 후속 단계이며 현재 문서의 주 범위는 상위 화면이다.

## 7. Minecraft 정책

- 현재 코드에 존재하더라도 메인 IA에는 편입하지 않는다.
- Campus 상위 허브에도 노출하지 않는다.
- 기능 삭제가 아니라 `정책 보류` 상태다.

## 9. 네이밍 원칙

사용자 노출 라벨은 아래를 기준으로 한다.

- `캠퍼스`
- `택시`
- `공지`
- `커뮤니티`
- `MY`

코드 레벨 route name은 구현 시 영어/한글 중 하나로 정리할 수 있지만, 사용자 노출 라벨은 위 명칭을 기준으로 유지한다.
