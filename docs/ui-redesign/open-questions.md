# Open Questions

작성일: 2026-03-10

## 1. 목적

이 문서는 아직 확정되지 않은 항목만 모아둔다.  
확정된 규칙은 다른 문서에만 기록한다.

## 2. 현재 보류 항목

## 2.1 글로벌 MY 진입 표준

확정된 것:

- MY는 하단 탭이 아니다.
- MY는 별도 화면이다.
- MY 화면에서는 bottom tab shell을 노출하지 않는다.
- MY 화면 좌측 상단에는 뒤로가기 버튼을 둔다.

미정:

- 모든 상위 화면 우상단 프로필 아이콘이 MY로 연결되는지
- 검색/알림/프로필 조합을 화면별로 다르게 둘지

## 2.2 상세 화면 리팩터링 시점

확정된 것:

- 현재는 상위 화면 우선 구현

미정:

- 상세 화면을 어떤 순서로 확장할지
- 공지 상세, 게시글 상세/작성, 택시 채팅/정산, 시간표 상세 중 우선순위

## 2.3 모션 수치

확정된 것:

- 과한 모션은 지양
- 짧고 조용한 전환 우선

미정:

- duration
- easing
- 탭/세그먼트/FAB/리스트 갱신의 세부 모션

## 2.4 배지 상한 표기

미정:

- unread badge가 두 자리 이상일 때 표기 정책
- 예: `9`, `15`, `99+` 중 어떤 방식을 쓸지

## 2.5 행사 카테고리 색 토큰

미정:

- 상위 Figma만으로 `행사` 라벨의 정확한 토큰 세트가 충분히 고정되지 않음

## 2.6 로딩 / 빈 상태 / 에러 시각 규칙

미정:

- skeleton 사용 여부
- empty state illustration 사용 여부
- inline error vs full-screen error 패턴

## 2.7 Campus 브랜드 영역 확장 방식

확정된 것:

- `SKURI` 워드마크는 SVG/이미지 자산으로 구현한다.

미정:

- 추후 로고/앱 아이콘을 함께 넣을 때 워드마크와 어떤 조합으로 배치할지
