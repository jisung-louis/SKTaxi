# 법적 리스크 분석 보고서

## 1. 여객자동차운수사업법 및 택시운송사업의 발전에 관한 법률

### 1.1 현재 조치 사항
- ✅ 약관에 "택시 운송의 주선, 알선, 중개 또는 배차 서비스가 아님" 명시
- ✅ 택시 기사와의 직접 연결 기능 없음 (코드 확인 완료)
- ✅ 외부 앱(카카오T, 티머니GO, Uber) 딥링크만 제공
- ✅ 실제 택시 호출은 외부 앱에서 처리

### 1.2 리스크 분석

#### ✅ 리스크 해소됨
1. **택시 주선/알선 여부**
   - **법적 요건**: 여객자동차운수사업법 제4조에 따르면, 택시 운송사업을 하려면 사업자등록 및 허가가 필요
   - **현재 상태**: 
     - 택시 기사와의 직접 연결 기능 없음
     - 택시 배차 기능 없음
     - 외부 앱으로만 유도 (딥링크)
   - **결론**: 택시 운송사업에 해당하지 않음. 리스크 해소됨.

2. **택시 운송 중개**
   - **법적 요건**: 택시운송사업의 발전에 관한 법률에 따른 중개사업 허가 필요
   - **현재 상태**: 
     - 사용자 간 매칭만 제공 (파티 생성/참가)
     - 택시 호출은 외부 앱에서 처리
     - 수수료/중개료 없음
   - **결론**: 택시 운송 중개사업에 해당하지 않음. 리스크 해소됨.

#### ⚠️ 잠재적 리스크 (낮음)
1. **"택시 동승 매칭" 용어 사용**
   - **리스크**: "매칭"이라는 용어가 운송 중개로 오해될 수 있음
   - **완화 조치**: 
     - 약관에 "커뮤니티 서비스"로 명시
     - "정보 제공" 서비스임을 강조
   - **권장사항**: 
     - 앱 내에서 "택시 동승 파티 모집" 또는 "동승자 찾기" 등으로 용어 변경 고려
     - 약관에서 "매칭"보다는 "정보 제공" 용어 사용 강화

2. **파티 생성/수락 기능**
   - **리스크**: 파티 수락 기능이 운송 중개로 해석될 수 있음
   - **현재 상태**: 
     - 단순히 채팅방 참가 허용 기능
     - 택시 호출과 무관
   - **결론**: 정보 제공 범위 내. 리스크 낮음.

### 1.3 최종 평가
**리스크 해소 여부**: ✅ **대부분 해소됨**

약관 명시와 실제 기능 구현이 일치하며, 택시 운송사업에 해당하지 않음이 명확함. 다만 용어 사용에 주의 필요.

## 2. 위치정보의 보호 및 이용 등에 관한 법률

### 2.1 현재 조치 사항
- ✅ `userLocations` 컬렉션 제거됨 (코드 확인 완료)
- ✅ 실시간 위치 추적 기능 제거됨
- ✅ 파티원 위치 공유 기능 제거됨
- ✅ 위치정보는 클라이언트 측에서만 사용 (가까운 파티 정렬)
- ✅ 개인정보 처리방침에 명시

### 2.2 리스크 분석

#### ✅ 리스크 해소됨
1. **개인위치정보 수집/저장**
   - **법적 요건**: 위치정보법 제15조 - 개인위치정보 수집 시 신고 필요
   - **현재 상태**: 
     - 사용자의 현재 위치를 서버에 저장하지 않음
     - `userLocations` 컬렉션 제거 확인
     - Firestore 인덱스에서도 제거됨
   - **결론**: 개인위치정보를 수집/저장하지 않음. 신고 의무 없음. 리스크 해소됨.

2. **개인위치정보 제공**
   - **법적 요건**: 위치정보법 제17조 - 제3자 제공 시 동의 및 신고 필요
   - **현재 상태**: 
     - 다른 사용자에게 위치 정보 제공하지 않음
     - 파티원 위치 표시 기능 제거됨
   - **결론**: 개인위치정보를 제3자에게 제공하지 않음. 리스크 해소됨.

3. **목적지 좌표 저장**
   - **법적 요건**: 목적지 정보는 "개인위치정보"가 아님
   - **법적 근거**: 위치정보법 제2조 제2호 - "개인위치정보"는 특정 개인의 위치에 관한 정보
   - **현재 상태**: 
     - 파티 생성 시 `departure`, `destination`의 좌표만 저장
     - 이는 목적지 정보이지 사용자의 현재 위치가 아님
   - **결론**: 개인위치정보가 아님. 문제 없음.

4. **클라이언트 측 위치 사용**
   - **법적 요건**: 단말기 내에서만 사용하는 경우 신고 불필요
   - **현재 상태**: 
     - `useCurrentLocation` 훅으로 클라이언트 측에서만 위치 획득
     - 가까운 파티 정렬에만 사용
     - 서버 전송 없음
   - **결론**: 위치정보법 적용 대상 아님. 리스크 해소됨.

### 2.3 최종 평가
**리스크 해소 여부**: ✅ **완전히 해소됨**

위치정보 관련 모든 리스크가 해소되었으며, 위치정보법상 신고 의무 없음.

  - 수수료/중개료 없음

- **현재 상태**:

  - 웹소켓은 React Native 기본 기능 (Firebase Firestore 실시간 구독)
  - P2P 직접 연결 없음
- **결론**: 서버를 통한 통신이므로 문제 없음. 리스크 없음.

## 6. 결론
