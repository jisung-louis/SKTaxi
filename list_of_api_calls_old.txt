  요약

  Firestore 통신 방식 (클라이언트 앱)
  ├── 읽기 (Read)
  │   ├── 실시간 구독: onSnapshot (29개) — 데이터 변경 시 자동 반영
  │   └── 일회성 조회: getDoc / getDocs (30+개) — 한 번 읽고 끝
  ├── 쓰기 (Write)
  │   ├── 단건 쓰기: addDoc / setDoc / updateDoc / deleteDoc (50+개)
  │   └── 배치 쓰기: writeBatch (4곳)
  └── 미사용
      ├── runTransaction (트랜잭션)
      └── httpsCallable (Cloud Functions 호출)

  참고로 "HTTP 호출"은 아닙니다. getDoc/getDocs 등도 내부적으로는 Firestore
  SDK의 gRPC/WebSocket 연결을 사용하며, REST API를 직접 호출하는 부분은
  프로젝트에 없습니다. 또한 Cloud Functions를 클라이언트에서 직접
  호출(httpsCallable)하는 패턴도 없고, Cloud Functions는 Firestore 트리거(서버
  측)로만 동작하고 있습니다.
